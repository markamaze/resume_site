Fudget Budget

Summary

Fudget Budget is a personal budgeting tool built for Android. Enter income & expense transactions and the frequency they occur, and those are projected out to calculate your future account balance with each transaction.

User Details

To get started, go to the Transactions tab and enter all regular income and expences. These are the main transactions which are used to project the budget.

When transactions are entered, go to the Budget tab. You will see your transactions projected out by date here. Each one will have a calculated balance at the end. Settings are available to adjust months projected and set a minimum balance threshold.

Any projected transactions due or overdue will be emboldened, and can be recorded. Future items highlighted show the projected balance is either below the threshold or below zero.

Each projected transaction can be modified as needed. This allows for making changes to assure a future balance doesn't go low, or to maximize current expendable income, or for whatever is needed to make your budget work.

If you work out your budget, then don't come back to the app for a while and find it way behind or inaccurate, you can easily fudge the budget and spend no time getting it ready for planning. In the settings of the Budget tab, you can get everything up to date in one step. Just check whether you want to record or delete all past due projections, and if you want to calculate or manually set your new current balance. Submit and your budget is instantly current.

Technical Details

The model for this program is built around the Transaction, from which ProjectedTransactions and RecordedTransactions are derived. With Transactions created, ProjectedTransactions are created out to number of months defined by a setting value. Those ProjectedTransactions are then displayed, by date, with a calculated balance displayed with it. When the date of a transaction is today or past, the user can "record" the projected transaction. The RecordedTransactions are stored, and summed together for the current balance.

Each ProjectedTransaction can be adjusted from the values it was derived from, and RecordedTransactions can also be altered if needed. Altered projections can be reset to their original values individually, by transaction, or for all transactions.

There is a setting for a balance threshold, which is used with the lowest projected balance in order to calculate the expendable funds given the current balance. If any projected balances are below the threshold, they will appear highlighted, and the expendable funds will be zero. This allows the user to easily see if their current budget will cause them to get a low balance, overdraw their account, or how much they can spend now without going below their threshold at any projected time.

The action to "fudge the budget", simply records or deletes all due/pastdue ProjectedTransactions. The current balance is still calculated the same, however if the user chooses to set a new balance, this is done by creating a RecordedTransaction for the difference between what is calculated and what the user sets the new balance to.
